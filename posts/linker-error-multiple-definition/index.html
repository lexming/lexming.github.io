The main errors from the linker are the multiple definition of 'symbol' errors, which are caused by duplicate LDFLAGS in the linker command. In this case, -lpthread is declared twice

g++ […] -ldl -lpthread -lrt -lm -lpthread -ljpeg -lpng […]

This error can be easily solved by removing the duplicate linker flag. However, depending on the underlying build system, this can be more or less involved. If the build is done with make, it can be just a matter of manually editing the corresponding Makefile.

In this case, the build is based on CMake and hence, it is necessary to dig into its modules to find the spot where this duplication happens. Once the origin of the duplication is found, CMake provides the REMOVE_DUPLICATES method to clean all duplicates from a list.

The following code snippet automatically removes duplicate elements in FOX_LIBRARY, which contains these flags in the build of SUMO (cmake_modules/FindFOX.cmake):

1
2
3

separate_arguments(FOX_LIBRARY)
list(REMOVE_DUPLICATES FOX_LIBRARY)
string(REPLACE ";" " " FOX_LIBRARY "${FOX_LIBRARY}")
